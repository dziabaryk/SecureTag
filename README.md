# ** CRM-система с интегрированным AI-ассистентом**

## **Краткое описание проекта**

Этот проект представляет собой разработку CRM-системы нового поколения, ориентированной на малый и средний бизнес (SMB) в сферах e-commerce, розничной торговли и услуг.

### **Основные особенности:**

1. **Интегрированный AI-ассистент:** Ключевая уникальная функция — использование технологий AI/ML для персонализированных подсказок и обучения пользователей *в процессе* работы с CRM. Это позволяет быстро адаптировать новых сотрудников и повышать эффективность опытных.  
2. **Управление отношениями с клиентами (CRM):** Стандартный функционал для ведения базы клиентов, истории взаимодействий, воронок продаж и аналитики.  
3. **Импортозамещение:** Цель проекта — предоставить конкурентоспособное отечественное решение, снижающее зависимость от крупных западных CRM-систем (Salesforce, HubSpot).

**Ожидаемый эффект (ТЭО):** Прогнозируемый срок окупаемости инвестиций составляет 2.5–3 года, что выше средней эффективности на рынке корпоративного ПО.

## **Инструкция по сборке и запуску (Предполагаемые шаги)**

Проект состоит из трех основных компонентов: Фронтенд, Бэкенд (API) и Сервис AI/ML.

### **1\. Предварительные требования**

* Node.js (v18+)  
* Python (v3.10+)  
* Docker (рекомендуется для быстрого развертывания БД)

### **2\. Настройка базы данных**

Для проекта используется PostgreSQL. Рекомендуется запустить его через Docker:

\# Запуск контейнера PostgreSQL  
docker run \--name crm-db \-e POSTGRES\_USER=user \-e POSTGRES\_PASSWORD=pass \-p 5432:5432 \-d postgres:latest

### **3\. Запуск Бэкенда (Django/Python)**

\# Переходим в директорию бэкенда  
cd backend/  
\# Создаем и активируем виртуальное окружение  
python \-m venv venv  
source venv/bin/activate  \# Для Linux/macOS  
\# source venv/Scripts/activate \# Для Windows  
\# Установка зависимостей  
pip install \-r requirements.txt  
\# Применение миграций и создание суперпользователя  
python manage.py migrate  
python manage.py createsuperuser  
\# Запуск сервера  
python manage.py runserver 8000

### **4\. Запуск Фронтенда (React)**

\# Переходим в директорию фронтенда  
cd ../frontend/  
\# Установка зависимостей  
npm install  
\# Запуск приложения  
npm start

Приложение будет доступно по адресу http://localhost:3000.

### **5\. Запуск Сервиса AI/ML**

Сервис AI запускается отдельно для обработки запросов ассистента.

\# Переходим в директорию AI-сервиса  
cd ../ai\_service/  
\# Установка зависимостей (в отдельном venv)  
pip install \-r requirements.txt  
\# Запуск сервиса (пример)  
python ai\_server.py

*(Убедитесь, что все необходимые API-ключи (например, для OpenAI или Google Gemini) прописаны в файлах .env.)*

## **Информация для разработчиков**

### **Используемые технологии**

| Компонент | Технология | Описание |
| :---- | :---- | :---- |
| **Фронтенд** | **React** (с TypeScript), Tailwind CSS | Клиентская часть приложения, обеспечивает SPA-интерфейс. |
| **Бэкенд** | **Python/Django** | RESTful API для взаимодействия с БД, бизнес-логика, авторизация. |
| **База данных** | **PostgreSQL** | Основное хранилище данных (пользователи, клиенты, сделки). |
| **AI/ML** | **Python** (библиотеки LLM, LangChain) | Сервис для обучения и генерации персонализированных подсказок. |
| **Стили** | **Tailwind CSS** | Быстрая и адаптивная верстка. |

### **Как начать работу**

1. **Клонирование репозитория:**  
   git clone \[ссылка\_на\_ваш\_репозиторий\]

2. **Настройка переменных окружения:**  
   * Скопируйте файлы .env.example во всех трех директориях (backend, frontend, ai\_service) и переименуйте их в .env.  
   * Заполните необходимые параметры, включая ключи БД и секретные ключи.  
3. **Локальный запуск:** Выполните шаги, описанные в разделе "Инструкция по сборке и запуску".  
4. **Ветвление:** Все новые фичи и исправления должны разрабатываться в отдельных ветках, основанных на develop (например, feature/add-ai-training). Слияние в main происходит только после успешного ревью и прохождения CI/CD.
