SecureTag Project: Система управления биометрическими трекерами

||||| Краткое описание проекта |||||

SecureTag — это инновационная система гео-трекинга, основанная на использовании компактных аппаратных трекеров, защищенных биометрической идентификацией. Основная задача проекта — обеспечение безопасности личных вещей и мгновенное оповещение владельца в случае утери или попытки несанкционированного доступа.

Уникальность: Использование биометрических хешей (отпечатков пальцев) для управления режимом блокировки устройства.

Эффективность (ТЭО): Прогнозируемый срок окупаемости инвестиций составляет 2.5–3 года, с ожидаемой доходностью 15–20% к третьему году.

||||| Инструкции по сборке и запуску |||||

Поскольку проект находится на этапе проектирования, ниже описаны предполагаемые шаги, необходимые для локального развертывания системы:

Клонирование репозитория:

git clone [https://github.com/dziabaryk/SecureTag/](https://github.com/dziabaryk/SecureTag/)
cd SecureTag-Project



Настройка окружения (Backend):

Установка Node.js (v18+) и Docker.

Сборка и запуск контейнера с PostgreSQL и Backend-сервисом: docker-compose up -d.

Запуск мобильного приложения (Frontend):

Переход в директорию /src/mobile-app.

Установка зависимостей: npm install.

Запуск эмулятора: npm run android или npm run ios.

||||| Информация для разработчиков |||||

Используемые технологии (Проектируемый стек):

Backend: Node.js, TypeScript, Express/NestJS.

Database: PostgreSQL (с расширением PostGIS для гео-данных).

Architecture: Microservices, Event-Driven (Pub/Sub) для гео-потока.

Mobile App: Kotlin Compose, Swift SwiftUI

VCS: Git Flow (основные ветки: main, develop, feature/*).

Как начать работу:

Ознакомьтесь с архитектурными диаграммами в /docs/lr3 и спецификацией API.

Создайте новую ветку от develop для своей задачи: git checkout -b feature/TASK-ID_your-feature.

Все конфигурационные файлы, содержащие секреты, должны быть получены от DevOps-команды и НЕ ДОЛЖНЫ быть закоммичены (см. .gitignore).
